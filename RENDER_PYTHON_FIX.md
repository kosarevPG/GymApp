# КРИТИЧЕСКИ ВАЖНО: Установка Python 3.11 на Render

## Проблема
Render по умолчанию использует Python 3.13, но `pydantic-core` для старых версий не имеет предкомпилированных wheels для Python 3.13 и пытается компилироваться из исходников (требует Rust), что не работает на Render.

## Решение: Явно указать Python 3.11

### Шаг 1: Обновите настройки Render

1. Зайдите в сервис `gym-logger-bot` на Render.com
2. Перейдите в **Settings**
3. Найдите секцию **Build & Deploy** или **Environment**
4. Найдите поле **Python Version** (может быть в разных местах в зависимости от версии интерфейса)

### Шаг 2: Установите Python 3.11

**Вариант A: Через поле Python Version**
- Если есть поле "Python Version", установите: `3.11.0`

**Вариант B: Через переменную окружения**
- В разделе **Environment Variables** добавьте:
  - Key: `PYTHON_VERSION`
  - Value: `3.11.0`

**Вариант C: Через runtime.txt (может не работать)**
- Файл `runtime.txt` уже содержит `python-3.11.0`
- Но Render может его игнорировать, поэтому лучше использовать Вариант A или B

### Шаг 3: Сохраните и перезапустите

1. Сохраните изменения
2. Нажмите **Manual Deploy** → **Deploy latest commit**
3. Проверьте логи - должно быть: "Installing Python version 3.11.0..."

## Проверка

После деплоя в логах должно быть:
```
==> Installing Python version 3.11.0...
==> Using Python version 3.11.0
```

А НЕ:
```
==> Installing Python version 3.13.4...
==> Using Python version 3.13.4
```

## Если не помогает

Если Render все еще использует Python 3.13, попробуйте:
1. Удалить сервис и создать заново с явным указанием Python 3.11
2. Или использовать другой хостинг (Heroku, Railway, Fly.io)
